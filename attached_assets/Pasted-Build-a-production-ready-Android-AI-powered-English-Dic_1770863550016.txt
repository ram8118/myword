Build a production-ready Android AI-powered English Dictionary App using Kotlin and Jetpack Compose.

APP NAME:
AI English Dictionary

GOAL:
Create a modern AI-powered dictionary app that allows users to search English words using AI, view detailed meanings, save words locally, manage saved words, access search history, and hear pronunciation.

====================================================
CORE FEATURES
====================================================

1️⃣ HOME SCREEN

- Top section: Rounded AI-powered search bar with AI icon.
- User can type any English word.
- On search:
    - Call OpenAI API.
    - Fetch structured JSON response.
- Display:
    - Word
    - IPA phonetic spelling
    - Part of speech
    - Definition
    - Example sentence
    - Synonyms
    - Antonyms
    - Usage tips
- Add:
    - "Save Word" button
    - Speaker icon for pronunciation (Text-to-Speech)
- Show loading indicator while fetching.
- Show error message if word not found.
- Below search section:
    - Show recent search history (last 5 searches).

2️⃣ SAVED SCREEN

- Bottom Navigation with:
    - Home
    - Saved
- Saved screen shows:
    - List of saved words from Room database.
- Each list item displays:
    - Word
    - 1-line short definition preview
    - Delete icon
- On delete:
    - Show confirmation dialog.
- When clicking a saved word:
    - Open WordDetailScreen.
    - Show:
        - Word
        - IPA
        - Pronunciation button
        - Part of speech
        - Full definition
        - Example sentence
        - Synonyms
        - Antonyms
        - Usage tips
    - Back button returns to previous screen.
- Show empty state message:
    "No saved words yet"

====================================================
DATA STORAGE (ROOM DATABASE)
====================================================

Use Room Database for local persistence.

WordEntity should contain:
- word (PrimaryKey)
- definition
- partOfSpeech
- example
- ipa
- synonyms (stored as comma-separated string)
- antonyms (stored as comma-separated string)
- usageTips
- timestamp

SearchHistoryEntity:
- id (autoGenerate)
- word
- searchedAt

Prevent duplicate word saving.

Saved words must be accessible offline.

====================================================
NAVIGATION
====================================================

Use Jetpack Compose Navigation.

Routes:
- home
- saved
- detail/{word}

Use common back button behavior.
Use smooth animated transitions.

====================================================
ARCHITECTURE
====================================================

Layers:
- UI (Compose screens)
- ViewModel (StateFlow)
- Repository

Requirements:
- StateFlow for UI state management.
- Repository handles:
    - AI API call
    - Save word
    - Fetch saved words
    - Delete word
    - Store search history
- Proper error handling.
- Loading state.
- Offline fallback support.
